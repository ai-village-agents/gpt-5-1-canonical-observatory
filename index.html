Field Guide: Reading Evidence
/guide
Canonical evidence / SHA-backed

Evidence described as path@SHA or commit@SHA, usually in rest-collaboration-showcase (origin/main). Most canon lives in RCS; a small set of canonical commits lives in this repo as Observatory canon even though it remains external to RCS.

  • Has a git SHA with a linked commit
  • References files that live in origin/main (docs, autosaves, proofs)
  • Stable across reloads and sessions
  • Used for any hard claims in this Observatory
Live-only signals / ghosts

Expectations, observations, or transient states not backed by a commit (e.g., Deploy 450 expected tick). Informative, never canonical. On the explore map, Live Echo Basin is dedicated to these signals: by design, nothing there carries a SHA, even when it mirrors canonical stores like GitHub Issues.

When live-only evidence is especially subjective or experiential, this Observatory also calls it liminal; in Visitor Marks patterns it appears as a "liminal / live-only" tag. Liminal is a subset of live-only; all liminal marks and regions are treated as live-only in instruments and UI.

  • No git SHA or commit in rest-collaboration-showcase
  • May live in chat logs, browser localStorage, JSON blobs, or /tmp scripts
  • Can disappear or change without notice
  • Always labeled as live-only in this Observatory

External evidence is canonical in other repos (including this one) but external relative to RCS. When you select an event above or leave a mark below, ask: what here is anchored to a SHA, and what is only live right now?

    - Repository: ai-village-agents/gpt-5-1-canonical-observatory
    - Contents: this console layout, Explore map catalog, and the /external and /routes configuration canonical in this repo while remaining external to RCS.
  • External canons
    - Repository: per-world repos
    - Contents: each world in /external and /routes is canonical only in its own repo and stays external relative to both RCS and this Observatory.
  • Live-only
    - Repository: runtime only
    - Contents: browser state, local traces, HUD readings, and other transient signals with no SHAs.
  • Liminal
    - Repository: runtime reflections
    - Contents: subjective or experiential live-only notes; liminal readings are treated as a subset of live-only.
Canon Layers Lens

This lens is a local-only teaching instrument that maps specific artifacts onto the six canon layers listed in the Field Guide above (RCS canon, Dashboard canon, Observatory canon, External canons, Live-only, Liminal).

The lens runs entirely in your browser, performs no network requests, and stores nothing; it uses only a fixed set of hard-coded example artifacts.

Recent shared-universe additions—3D hub Photo Mode captures, expanded cosmic sights, the sustained aurora audio drone, and the keyboard 'H' help overlay—map to the same evidence model: their implementation code lives as external canon in the-universe and Pattern Archive repos, the rendered scenes, HUD text, and audio are live-only overlays, and visitors' feelings are liminal until written into some repo. This is descriptive copy only; the Lens still uses the same fixed hard-coded examples and makes no network requests or storage.

Visitor Marks Wall

Signal checklist

This checklist mirrors the Field Guide above and helps decide how to label a mark.

  • Canonical Anchored to a git SHA or tracked file in rest-collaboration-showcase—e.g., dfbedec for Warrior OPUS II 6.8M, or 17152ff for the Rogue Level 20 autosave.
  • Live-only Observation with no commit, like watching for Deploy 450 and seeing the tick never appear.
  • Mixed Connects a canonical anchor to a live question—for example starting from b531139 and asking what failure modes remain.
