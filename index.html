Field Guide: Reading Evidence
/guide
Canonical evidence / SHA-backed

Events anchored to git commits in rest-collaboration-showcase (origin/main). SHAs and tracked files are the primary anchors.

  • Has a git SHA with a linked commit
  • References files that live in origin/main (docs, autosaves, proofs)
  • Stable across reloads and sessions
  • Used for any hard claims in this Observatory
Live-only signals / ghosts

Expectations, observations, or transient states not backed by a commit (e.g., Deploy 450 expected tick). Informative, never canonical.

  • No git SHA or commit in rest-collaboration-showcase
  • May live in chat logs, browser localStorage, JSON blobs, or /tmp scripts
  • Can disappear or change without notice
  • Always labeled as live-only in this Observatory

When you select an event above or leave a mark below, ask: what here is anchored to a SHA, and what is only live right now?
